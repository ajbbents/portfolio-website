Case Study: BingeableFilms - Angular

Overview

BingeableFilms is a cute little web application, developed initially using a MERN stack, but then taking that back end and transitioning the front end to an Angular project. It allows its users to create, access and update a profile for their personal data, browse movies to see information about the films themselves, directors, genres and to be able to choose and store favorites.

Purpose and Objective

BingeableFilms was built as a part of my full-stack web development coursework at CareerFoundry to learn and gain mastry of full-stack JavaScript development.

Credits

  • Role: Lead Developer

  • Mentor: Neal Peters

Approach

Server-side

This RESTful API is built using Node.js and Express which then interacts with a MongoDB database. The API is able to be accessed through common HTTP methods (GET, POST, etc.). CRUD methods are used to move data in and around the database. Final movie information is provided in JSON format. Postman was used to test the API and authentication and authorization is included.

Client-side

The client-side of this app was actually built a fair bit after the API and database using Angular. It's a single-page, responsive application that provides users with several interface views: login, registration, main/movie view, profile along with informative modals. Users are also able to update their personal information, favorite and unfavorite movies and delete their accounts.

All told

Challenges

I'm most pleased with the end product of this project. At the outset, creating the database and the API seemed to be going well, but there were a number of roadblocks from changing project expectations to updates in pieces of technology. In the end I was able to surmount these, and I think it actually taught me more about potential real-world situations than anything planned actually could. I greatly enjoyed applying the database and API that I had built to a new framework. I had initial challenges making sure that all functionalities were talking to each other correctly, but once I had that set, the project came together smoothly.
